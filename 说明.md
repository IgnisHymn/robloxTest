下面是为 Roblox 环境配置 jecs（TypeScript 版）的完整操作链，我会将这个过程拆解为几个清晰的步骤。它能帮你把整个流程梳理清楚。

### 第一步：准备本地开发环境

在开始编码前，我们首先需要在本地电脑上安装好所有必要的工具。

| 类别 | 所需工具 | 简述与作用 |
| :--- | :--- | :--- |
| **核心环境** | **Node.js** | 运行 TypeScript 编译器和 npm 包管理器的基础环境。建议安装最新的 LTS 版本。 |
| **代码编辑器** | **Visual Studio Code (VS Code)** | 编写和调试代码的现代化编辑器。它强大的扩展生态是进行高效开发的关键。 |
| **同步桥梁** | **Rojo 命令行工具** | 安装在电脑上的程序，用于监听项目文件变动并启动同步服务器。 |
| **同步桥梁** | **Rojo Studio 插件** | 安装在 Roblox Studio 中的插件，用于连接到命令行工具创建的服务器，实现文件双向同步。 |
| **包管理器** | **Wally** | 管理项目依赖的包管理器。你可以把它理解为 Roblox 生态中的“npm”。 |

#### 💡 工具安装小贴士

*   **Node.js**：直接从 [Node.js 官网](https://nodejs.org) 下载并安装即可。
*   **VS Code**：同样从 [VS Code 官网](https://code.visualstudio.com) 下载安装。
*   **Rojo**：
    1.  **命令行工具**：可以通过 [Rokit](https://rojo.space/)（推荐，后面会介绍）或从 [GitHub Releases](https://github.com/rojo-rbx/rojo/releases)下载预编译文件并放到你的系统 PATH 中。
    2.  **Studio 插件**：在 Roblox 的插件商店中搜索 "Rojo" 并安装，或直接访问其 [插件主页](https://www.roblox.com/library/1997686364/) 获取。
*   **Wally**：
    *   **💎 推荐：安装工具链管理器 Aftman**。这是最推荐的现代方法。Aftman 可以统一管理 `wally`, `rojo` 等多种工具，避免手动配置环境的麻烦。
        1.  访问 Aftman 的 [GitHub Releases 页面](https://github.com/LPGhatguy/aftman/releases) 下载并安装。
        2.  打开终端，使用 Aftman 安装工具：`aftman add UpliftGames/wally` 和 `aftman add rojo-rbx/rojo`。
        3.  运行 `aftman install`，Aftman 就会把这两个工具安装到你的项目目录下，实现环境隔离，不影响系统其他部分。

---

### 第二步：创建和初始化项目

环境准备好后，我们就在电脑上创建一个新的 Roblox 项目文件夹，并完成项目配置。

1.  **创建文件夹并打开终端**
    在你喜欢的位置创建一个项目文件夹（例如 `my-ecs-game`），并在 VS Code 中打开这个文件夹。

2.  **初始化 `roblox-ts` 项目**
    在 VS Code 的终端中运行以下命令。它会为你生成一个配置好的 TypeScript 项目骨架。
    ```bash
    npm init roblox-ts
    ```
    在设置过程中，通常会让你选择一个模板，对于新项目，选择 `game` 模板即可。这个命令会创建 `tsconfig.json`（TS配置）、`default.project.json`（Rojo配置）、`package.json`（npm包管理）和 `src`（源码）文件夹。

3.  **初始化 Wally 并安装 jecs**
    在项目根目录下，运行以下命令创建 `wally.toml` 配置文件：
    ```bash
    wally init
    ```
    然后，在生成的 `wally.toml` 文件中的 `[dependencies]` 部分，添加 jecs 依赖：
    ```toml
    [dependencies]
    jecs = "ukendio/jecs@0.6.0"  # 建议访问其 [Wally页面](https://wally.run/package/ukendio/jecs) 确认最新版本号
    ```
    添加完成后，运行 `wally install`，Wally 就会自动下载并将 jecs 包安装到你的 `Packages` 文件夹中。

---

### 第三步：将本地项目与 Roblox Studio 关联

现在项目已经配置好，我们需要把它“传”到 Roblox Studio 里。这里有两种主要的工作方式：

| 方式 | 简述与特点 |
| :--- | :--- |
| **📤 打包导入** | **一次性操作**：将整个项目打包成一个 `.rbxlx` 文件，然后在 Studio 中打开。 |
| **🔄 实时同步 (推荐)** | **持续性工作流**：启动 Rojo 服务器，并在 Studio 中连接。此后，你对本地文件的任何修改都会即时同步到正在运行的游戏中，是实现高效迭代的推荐方式。 |

#### **方式一：一次性打包导入**
这种方式适合测试或项目初始化。在项目终端运行以下命令，Rojo 会将你的项目编译成一个 `.rbxlx` 文件，然后用 Roblox Studio 打开它即可。
```bash
rojo build -o build.rbxlx
```

#### **方式二：实时同步 (核心工作流)**

1.  **启动 Rojo 服务器**
    在项目终端的项目根目录下，运行 Rojo 的命令行工具来启动同步服务器：
    ```bash
    rojo serve
    ```
    成功启动后，你会在终端看到类似 `Rojo server listening on port: 34872` 的信息。

2.  **在 Roblox Studio 中连接**
    打开 Roblox Studio，在顶部菜单栏的 "Plugins" 选项卡中找到并点击 "Rojo" 按钮。在弹出的 Rojo 面板中，点击 **Connect** 按钮。一旦连接成功，你本地项目中的 `src` 文件夹就会被镜像为 Studio 里的一个名为 `src` 的模块。

### 第四步：编写和运行你的代码

一切准备就绪，让我们动手写第一行 ECS 代码。

1.  **编写示例脚本**
    在 VS Code 中，进入 `src` 文件夹，新建一个服务脚本，例如 `src/server/main.server.ts`。在里面写入以下代码：
    ```typescript
    import { World } from "@rbxts/jecs";

    // 1. 创建一个世界
    const world = new World();

    // 2. 定义两个数据组件
    const Position = world.component<Vector3>();
    const Velocity = world.component<Vector3>();

    // 3. 创建一个实体，并赋予它位置和速度组件
    const player = world.entity();
    world.set(player, Position, new Vector3(0, 0, 0));
    world.set(player, Velocity, new Vector3(1, 0, 0));

    // 4. 在游戏循环中更新实体
    game.getService("RunService").heartbeat.Connect((dt) => {
        for (const [id, pos, vel] of world.query(Position, Velocity)) {
            // 更新：新位置 = 旧位置 + 速度 * 时间
            const newPos = pos.add(vel.mul(dt));
            world.set(id, Position, newPos);
            print(`Player moved to: ${newPos.X}, ${newPos.Y}, ${newPos.Z}`);
        }
    });
    ```

2.  **编译与同步**
    为了让 TypeScript 代码能被 Roblox 执行，我们需要一个编译步骤。再次打开一个终端（可以使用 VS Code 的分屏功能），运行：
    ```bash
    npm run watch
    ```
    这个命令会启动 TypeScript 编译器并进入“监听模式”，只要你的 `.ts` 文件有变动，它就会自动重新编译。编译后的 Luau 代码会输出到 `out/` 目录，而 Rojo 则会自动将这些文件同步到 Roblox Studio 中的 `src` 模块里。

至此，完整的 jecs 工作流程就打通了。一开始设置环境可能需要花些时间，但一旦跑通，后续写游戏、测试、反复迭代都会变得非常顺畅～
